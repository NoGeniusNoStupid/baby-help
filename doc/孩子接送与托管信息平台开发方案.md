# 孩子接送与托管信息平台开发方案

## 1. 背景与目标

本方案基于原始需求中的页面结构与交互形态进行借鉴，但业务定位已完全重构，不再与宠物服务相关。  
我们即将开发的是一个集孩子接送、托管班信息于一体的本地生活服务信息平台，致力于为用户提供需求发布与信息展示服务，帮助供需双方高效对接。

## 4. 新业务定位与边界

### 4.1 平台定位

本地“孩子接送 + 托管班信息”供需对接平台，核心是需求发布与信息展示，帮助家长和服务方快速触达。

### 4.2 核心角色

- 家长：发布需求、查找服务
- 服务方：托管班机构、接送服务提供者
- 平台运营：审核、治理、运营

### 4.3 核心场景

- 临时接送
- 固定接送
- 课后托管
- 寒暑假托管
- 就近比价
- 快速联系

### 4.4 核心价值

- 家长侧：省时、可信、可比选
- 服务方：稳定获客、精准触达
- 平台侧：可治理、可运营、可沉淀本地供给

## 5. 新版信息架构建议

- 首页：发布需求、信息大厅、热门托管班、同城滚动需求、认证推荐
- 发现：托管班专题、区域榜单、价格区间、特色标签
- 消息：系统通知、报名反馈、联系提醒、审核结果
- 我的：个人资料、我的需求、我的收藏、联系记录、设置与客服

## 6. MVP 功能清单（第一阶段）

### 6.1 用户与身份

- 微信授权登录
- 家长与服务方角色识别
- 基础实名认证（手机号与基础信息）

### 6.2 需求发布

- 接送与托管需求模板
- 时间地点
- 预算范围
- 人数与年级
- 备注信息
- 急单标识

### 6.3 信息大厅

- 区域筛选
- 时间筛选
- 预算筛选
- 卡片化展示
- 报名或联系入口

### 6.4 托管班信息展示

- 机构基础信息
- 服务时段
- 价格区间
- 服务范围
- 服务标签
- 图文介绍

### 6.5 供需触达

- 一键报名
- 站内消息通知
- 需求状态流转（待响应、沟通中、已完成、已关闭）

### 6.6 平台治理

- 内容审核
- 敏感词过滤
- 举报入口
- 黑名单机制

## 7. 二阶段增强方向

- 机构资质认证与信用分
- 智能匹配推荐（距离、时间、预算、标签）
- 交易能力增强（如后续引入担保机制）
- 城市专题与活动运营位
- 完成后双向评价体系

## 8. 关键业务流程

### 8.1 家长流程

发布需求 → 等待响应 → 筛选沟通 → 选择服务方 → 完成与评价

### 8.2 服务方流程

浏览大厅 → 报名或联系 → 沟通确认 → 提供服务 → 获取评价

### 8.3 平台流程

内容审核 → 异常监控 → 投诉处理 → 数据运营优化
